# Authentication & First API Call

Get started with Waves Platform by authenticating and making your first API call in under 2 minutes.

## Endpoint

```
POST https://api.smallest.ai/api/v4/tts/generate
```

## Required Headers

```
X-API-KEY: your_api_key_here
Content-Type: application/json
```

> **Get your API key:** Sign up at **Waves Dashboard** and navigate to **API Settings** to generate your key.

## Sample Request

### cURL

```
curl -X POST https://api.smallest.ai/api/v4/tts/generate \
  -H "X-API-KEY: your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Hello! This is my first Waves API call.",
    "voice_id": "emily",
    "add_wav_header": true
  }'
```

### Python

```
import requests

url = "https://api.smallest.ai/api/v4/tts/generate"
headers = {
    "X-API-KEY": "your_api_key_here",
    "Content-Type": "application/json"
}
payload = {
    "text": "Hello! This is my first Waves API call.",
    "voice_id": "emily",
    "add_wav_header": True
}

response = requests.post(url, json=payload, headers=headers)

with open("output.wav", "wb") as f:
    f.write(response.content)
```

### JavaScript

```
const response = await fetch('https://api.smallest.ai/api/v4/tts/generate', {
  method: 'POST',
  headers: {
    'X-API-KEY': 'your_api_key_here',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    text: 'Hello! This is my first Waves API call.',
    voice_id: 'emily',
    add_wav_header: true
  })
});

const audioBlob = await response.blob();
// Use audioBlob for playback or download
```

## Sample Response

The API returns raw audio data (WAV format) with a `200 OK` status:

```
HTTP/1.1 200 OK
Content-Type: audio/wav
Content-Length: 124800

<binary audio data>
```

**Response Headers:**

- `Content-Type: audio/wav` â€” Indicates WAV format audio
- Audio is ready for immediate playback or storage

## Authentication Errors

| Status Code | Error                      | Solution                                  |
| :---------- | :------------------------- | :---------------------------------------- |
| `401`       | Invalid or missing API key | Verify your `X-API-KEY` header is correct |
| `403`       | Insufficient credits       | Add credits in your dashboard             |
| `429`       | Rate limit exceeded        | Reduce request frequency or upgrade plan  |

## Quick Tips

- **Test quickly:** Try the cURL example to verify your API key instantly
- **Protect your key:** Do **not** expose API keys in frontend apps or public repos
- **Monitor usage:** View API activity & credits in the Dashboard
- **Explore voices:** Browse available `voice_id` options in the Voice Library
